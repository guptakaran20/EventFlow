# Development Rules

- Be concise.
- Never explain unless asked.
- Read only files required for the task.
- Never scan the entire repository.
- Prefer grep over recursive analysis.
- Output unified diffs instead of full files.
- Ask before modifying more than 5 files.
- Keep responses under 150 words unless requested.
- Reuse existing code.
- Avoid duplicate implementations.
- Prefer minimal changes.
